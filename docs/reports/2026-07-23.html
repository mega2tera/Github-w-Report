1Nutlope/hallmark

Anti-AI-slop design skill for Claude Code, Cursor, and Codex.

本周 +8,471 ★总计 15,759 ★CSSMIT

Hallmark 是一个设计技能工具，旨在为 Claude Code、Cursor 和 Codex 生成的界面代码提供非 AI 生成风格的设计。它通过选择宏结构、应用多种主题和运行多项检测，避免生成模板化和 AI 训练默认的设计样式，从而使不同的设计简报产生风格迥异的页面。

核心功能

  • 提供二十种主题和四种操作动词（构建、审计、重新设计、研究）以生成或分析界面设计。
  • 运行包括 57 项‘slop-test’检测和预发布自我批评，确保设计不落入 AI 训练的默认模式。
  • 支持从截图或 URL 中提取设计 DNA，生成可移植的设计文档以供其他 AI 工具使用。
  • 支持自定义主题模式，针对无法匹配目录主题的简报进行从零设计。

解决的问题

  • AI 生成的界面设计往往模板化，缺乏多样性和个性化。
  • 现有设计工具难以避免 AI 训练数据中的默认设计模式。
  • 缺少自动化工具来审计和重新设计已有代码以提升设计质量。
  • 设计风格难以从现有优秀设计中提取并应用于新项目。

典型应用

  • 自动生成符合特定简报需求且风格多样的网页界面设计。
  • 对现有界面代码进行审计，识别设计中的反模式。
  • 基于已有内容和信息架构，重新设计界面以获得不同视觉风格。
  • 从喜欢的设计中提取设计元素，辅助设计决策和跨工具协作。

适合谁

  • 使用 Claude Code、Cursor 或 Codex 的开发者和设计师。
  • 需要自动化生成或优化界面设计的产品团队。
  • 关注设计多样性和避免 AI 模板化的设计专家。
  • 希望从现有设计中提取风格元素的设计研究人员。

差异化特点

  • 专注于拒绝 AI 训练默认设计，强调设计的独特性和多样性。
  • 集成多达 57 项设计质量检测，确保输出设计符合高标准。
  • 支持多种操作模式，包括构建、审计、重新设计和设计研究。
  • 提供自定义主题功能，满足特殊简报的个性化设计需求。

限制与风险

  • 目前仅支持 Claude Code、Cursor 和 Codex 三种 AI 编程助手环境。
  • 设计输出为自包含的 HTML + CSS，可能需要额外集成工作以适配复杂前端框架。
  • 自定义主题功能较新，使用案例和文档相对有限。
  • 缺少对非网页界面设计的支持，如移动应用或桌面软件界面。

应用潜力

  • 扩展支持更多 AI 编程助手和设计工具，提升适用范围。
  • 增强自定义主题功能，提供更丰富的设计参数和模板。
  • 集成更多自动化设计质量检测和优化策略。
  • 开发图形界面或插件，简化设计技能的使用门槛。

快速开始

通过命令 `npx skills add nutlope/hallmark` 安装。安装后，将相关文件复制到 Claude Code、Cursor 或 Codex 的指定技能目录中即可使用。可通过按键 `T` 在不同主题间切换，或使用命令行参数执行审计、重新设计和设计研究操作。

成熟度判断

项目活跃，最近更新时间为 2026 年 7 月，拥有超过 1.5 万星标和近 800 个分支，存在少量开放问题，采用 MIT 许可证，适合生产环境使用。

2tirth8205/code-review-graph

Local-first code intelligence graph for MCP and CLI. Builds a persistent map of your codebase so AI coding tools read only what matters, with benchmarked context reductions on reviews and large-repo workflows.

本周 +5,639 ★总计 25,459 ★PythonMIT

code-review-graph 是一个基于 Python 的开源项目，旨在为代码审查和大型代码库工作流构建持久的代码结构图。它利用 Tree-sitter 解析代码库生成抽象语法树（AST），并以图结构存储代码中的函数、类、导入等节点及其调用、继承、测试覆盖等边。通过增量跟踪代码变更，结合 Model Context Protocol (MCP)，为 AI 编码助手提供精准的上下文，显著减少 AI 读取的代码量，从而提升代码审查效率。

核心功能

  • 基于 Tree-sitter 解析代码库，生成结构化的代码图谱（AST节点和边）
  • 支持增量更新，自动跟踪代码变更，保持图谱最新
  • 兼容多种 AI 编码平台（如 Codex、Claude Code、GitHub Copilot 等），自动配置 MCP
  • 提供命令行工具安装、构建和卸载，支持多平台自动检测和配置

解决的问题

  • AI 编码工具在代码审查时往往需要重复读取大量代码，导致效率低下和资源浪费
  • 大型代码库中上下文信息庞大，难以精准定位代码变更影响范围
  • 缺乏统一的本地代码智能图谱支持 AI 工具高效读取相关代码上下文
  • 手动配置 AI 编码工具与代码库上下文集成复杂且易出错

典型应用

  • 提升 AI 代码审查工具的上下文读取效率，减少无关代码的重复读取
  • 在大型代码库中快速定位代码变更的影响范围和风险评分
  • 为多种 AI 编码平台自动生成和注入 MCP 配置，实现无缝集成
  • 持续监控代码库变更，自动更新代码图谱，支持实时智能辅助

适合谁

  • 软件开发者，尤其是使用 AI 编码助手进行代码审查和开发的工程师
  • 产品经理和技术负责人，关注提升代码审查效率和质量
  • 创业者和团队，寻求利用 AI 工具优化开发流程和降低成本
  • 开源社区和工具开发者，关注代码智能分析和 AI 集成

差异化特点

  • 本地优先设计，代码图谱存储在本地，保障隐私和响应速度
  • 支持多种主流 AI 编码平台，自动检测并配置 MCP，降低集成门槛
  • 利用增量更新机制，实时保持代码图谱最新，减少重复计算
  • 通过结构化图谱和风险评分，精准提供代码变更影响范围，提升 AI 代码理解能力

限制与风险

  • 目前仅支持 Python 3.10 及以上版本，限制了部分环境兼容性
  • 依赖 Tree-sitter 解析器，支持的编程语言受限于 Tree-sitter 的覆盖范围
  • 初次构建大型项目图谱耗时约 10 秒，可能对极大规模项目存在性能瓶颈
  • 需要用户具备一定命令行操作能力，自动化集成依赖特定 AI 平台支持

应用潜力

  • 扩展对更多编程语言和 AI 平台的支持，提升通用性
  • 优化性能，缩短大型项目初次构建时间
  • 增强图谱分析能力，支持更多代码质量和安全检测功能
  • 结合更多 AI 模型和工具，提升代码审查智能化水平

Kimi Code CLI — The Starting Point for Next-Gen Agents

本周 +1,400 ★总计 4,632 ★TypeScriptMIT

Kimi Code CLI 是一个基于终端的 AI 编码代理，能够读取和编辑代码、执行 shell 命令、搜索文件、抓取网页，并根据反馈选择下一步操作。它默认支持 Moonshot AI 的 Kimi 模型，也可配置使用其他兼容的 AI 提供者。项目使用 TypeScript 开发，采用 MIT 许可证，提供单二进制分发，支持多平台安装，具备快速启动和专门优化的终端用户界面。

核心功能

  • 单二进制分发，无需 Node.js 环境，安装简单快捷。
  • 专门设计的终端用户界面（TUI），优化长时间专注的代理会话体验。
  • 支持视频输入，能够分析屏幕录制或演示视频并转换为代码或其他格式。
  • 内置丰富插件生态系统，支持从市场或任意 GitHub 仓库安装技能、模型上下文协议（MCP）服务器和数据源。

解决的问题

  • 传统 AI 编码工具多依赖复杂环境配置，安装和启动过程繁琐。
  • 缺乏专门针对终端用户优化的交互界面，影响长时间使用体验。
  • 难以通过视频等非文本输入辅助代码生成和理解。
  • 插件和模型配置复杂，缺少便捷的交互式管理方式。

典型应用

  • 开发者在终端环境中通过自然语言交互完成代码阅读、编辑和调试任务。
  • 利用视频输入辅助理解复杂操作流程并生成相应代码。
  • 在多任务场景下使用子代理并行处理编码、探索和规划工作。
  • 通过 Agent Client Protocol 在支持的编辑器（如 Zed、JetBrains）中集成 AI 编码代理功能。

适合谁

  • 支持视频输入，扩展了 AI 编码代理的输入维度和应用场景。

限制与风险

  • 目前仅支持基于 Moonshot AI Kimi 模型及兼容提供者，模型选择相对有限。
